About

This project aims to use data from Twitter to get to know the popularity of the most important database engines, according the March 2019 version of the DB-Engines Ranking. These engines are:

  • Oracle Database
  • MySQL
  • SQLServer
  • PostgreSQL
  • MongoDB
  • IBM DB2
  • Microsoft Access
  • Redis
  • Elasticsearch
  • SQLite

The main interest is to be able to locate each tweet in a map, according to the tweet's or the user's location and to the topic (database engine) they refer to.

For this purpose, Kafka and Spark Streaming are used to process, transform and store data in Elasticsearch and MongoDB databases. Besides, Leaflet.js is the framework the map is developed with.

All the app is deployed with Node.js, alongside with the API where registered users can make their own requests to get the stored data. You can know more about the API here.

You can find all the code of the project and the instructions to run it on your own on this GitHub repo. Feel free to customize the Twitter topics or hashtags you want to filter the tweets by.
